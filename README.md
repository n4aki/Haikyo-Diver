# 廃墟ダイバー

崩壊した研究施設を探索し、酸素が尽きる前に階段を見つけて降下していく、ターン制2Dグリッド型ローグライクの最小プロトタイプです。  
素の HTML / CSS / JavaScript だけで動作し、GitHub Pages にそのまま配置できます。

## ゲーム概要

- 複数の部屋と通路で構成された広いランダムフロアを探索
- プレイヤーは 1 ターンに 1 マスだけ上下左右へ移動、または待機
- 2 行動ごとに酸素が 1 減少し、0 になると毎ターン HP が減少
- 敵はターン制で動き、隣接すると攻撃
- 階段を踏むとアップグレードを1つ選び、次フロアへ進行
- 5F / 10F はボスフロア
- 10F のボスを倒すと仮クリア

## ローカルでの起動方法

ビルドは不要です。以下のどちらかで動かせます。

1. `index.html` をブラウザで直接開く
2. ルートディレクトリで簡易サーバーを立てる

```bash
# Python 3 がある場合
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000` を開いてください。

## GitHub Pages で公開する手順

1. このディレクトリを GitHub リポジトリへ push する
2. GitHub のリポジトリ設定で `Settings` → `Pages` を開く
3. `Build and deployment` の `Source` を `Deploy from a branch` にする
4. Branch に `main`、フォルダに `/ (root)` を指定して保存
5. 数分待つと公開 URL が発行される

`index.html` をルートに置いているので、そのまま GitHub Pages で動作します。

## 現時点の仕様

- マップは毎フロアごとにランダム生成
- ワールド全体は 26x26、表示は 12x12 前後のビューポートでスクロール表示
- 1フロアは 4〜8 部屋前後の矩形部屋と通路で構成
- すべての部屋は通路で接続され、開始地点から階段まで到達可能
- 開始部屋と階段部屋はできるだけ離れた位置になるよう選出
- 部屋には `通常部屋` `物資部屋` `危険部屋` の内部タイプがあり、中身の傾向が変化
- 通常フロアの合間にボス専用の単一大部屋フロアが挟まる
- プレイヤー初期値
  - HP: 12
  - 酸素: 18
  - 攻撃力: 3
- 敵は複数種類
  - `スカウト (s)`: 基本の近接敵
  - `ブルート (B)`: 遅いが硬い重装敵
  - `ラッシャー (R)`: 2マス詰めてくる高速突進敵
  - `スナイパー (G)`: 直線上から射撃する遠距離敵
  - `バースト (X)`: 接近時や撃破時に周囲へ爆発する自爆敵
  - `メディク (M)`: 周囲の敵を回復する支援敵
  - `ウォーデン (W)`: ボスフロアに出現する高耐久ボス
- 通常フロアの敵はフロア帯ごとに出現傾向が変化
  - `序盤 (1-2F)`: スカウト / ラッシャー / ブルート中心
  - `中盤 (4F)`: 序盤構成にスナイパーが加わる
  - `後半 (5F以降の通常フロア)`: スナイパーに加えてバーストとメディクも出現
- アイテムは 2 種類
  - `応急キット (+)`: HP を 5 回復
  - `酸素カートリッジ (O)`: 酸素を 7 回復
- プレイヤー、敵、アイテム、階段は基本的に部屋内に配置
- 通路は移動専用で、階段やアイテムは基本的に配置されない
- 開始部屋は安全寄り、遠い部屋ほど敵が増えやすい軽い偏りあり
- 部屋ごとに敵やアイテムの偏りがあり、探索の優先度を考えやすい
- 物資部屋はアイテムが出やすく、危険部屋は敵が多く強敵が出やすい
- 未探索タイルは暗く、既探索タイルと現在視界内タイルで表示が変化
- 通路では視界が狭く、部屋に入るとその部屋全体が見える
- 敵とアイテムは現在視界内にいるときだけ表示される
- マップはプレイヤー移動に追従してスクロールし、HUD とログは固定表示
- スマホでは画面下の仮想ボタン、PC では矢印キー / WASD / スペースで操作
- 敵への攻撃は、敵のいるマスへ移動しようとしたときに実行
- フロア突破時に 3 択のラン中アップグレードを選択可能
- ボス撃破時は全回復し、通常より大きい 4 択報酬を獲得
- 選んだアップグレードは HUD から簡易確認可能
- 死亡するとゲームオーバー、リスタート可能
